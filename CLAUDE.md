# abm — сайт сообщества практиков ABM NeuroMovement

Статический одностраничный сайт. Хостинг: **GitHub Pages**, ветка `main` → домен **abminfo.ru** (CNAME). Сборки нет — что в `main`, то и на сайте. Push в `main` выкатывается автоматически за ~1–2 минуты.

Типичная задача владельца (Алексей) с телефона: «убери город X у практика Y», «поменяй телефон у Z», «добавь практика». Делаешь правку → показываешь дифф → по его «пушь» коммитишь и пушишь.

## Данные практиков — два места, НЕ одно

> ⚠️ **Важно (исправлено 2026-06-05):** раньше здесь было написано, что `index.html` строит карточки сетки из JSON. Это **неверно** и приводило к багу «добавил практика в JSON — карточка не появилась». Сетка верстается вручную.

Данные практика живут в **двух местах**, и при добавлении/удалении надо трогать оба:

1. **`practitioners.json`** — питает **модалку** (детали по клику) и **аккордеон городов** (`initCitiesAccordion` строит его из JSON по полю `city`). Ключ — `<slug>`.
2. **Сетка карточек в `index.html`** — **статический HTML**, блок `<div class="pgrid">` в секции `#search`. Каждая карточка — `<article class="pcard" data-id="<slug>">` с фото-фоном, именем, городом (и `<div class="away">Выезд: …</div>` если есть). `data-id` обязан совпадать со slug в JSON — по нему модалка находит данные. Карточка без фото — добавляй класс `pcard empty` и без `<div class="ph">`.

**Добавить практика:** (а) запись в `practitioners.json`; (б) `<article class="pcard">` в `pgrid`; (в) фото в `images/practitioner-photos/`. Новый город сам подтянется в аккордеон из JSON, но **в сетке карточка не появится без шага (б)**. Неактивных практиков (напр. Чугунов) в сетку не кладут — оттого число записей в JSON может быть больше числа карточек.

**Кнопка «Сайт / Telegram»** (`open()` в `index.html`) сама вытаскивает одну рабочую ссылку из `website`: первый `http(s)`-URL из списка, либо `@handle` → `t.me/handle`, либо префикс `https://`; значения `-` / `нет` / пусто → кнопки нет. Можно класть «грязное» поле с несколькими ссылками — отрендерит первую.

Формат JSON: объект `{ "<slug>": { …поля… }, … }`. Поля одной записи:

| поле | смысл |
|---|---|
| `name` | имя практика |
| `city` | основной город приёма (может быть «Город / Город» или с пометкой) |
| `address` | адрес |
| `phone`, `email`, `website` | контакты (строки; `"-"`, `"нет"`, `null` = нет) |
| `home` | приём на дому — `"Да"` / `"Нет"` |
| `travel` | выезд — `"Да"` / `"Нет"` |
| `travel_cities` | список выездных городов, разделители — запятая и/или перенос строки |
| `bio` | текст о практике (могут быть `\n`) |
| `photo` | имя файла в `images/practitioner-photos/` или `null` |

**Семантика «убрать город»:** чаще всего речь про `travel_cities` (выездной список). Если город совпадает с основным `city` — уточни, что именно убрать. Не угадывай молча.

## Жёсткие правила перед коммитом/пушем

Сайт **живой и клиентский** (контакты реальных людей), push в `main` = моментальный выкат. Поэтому:

1. **Валидация JSON.** После любой правки `practitioners.json` — прогони `jq empty practitioners.json`. Если не проходит (сломана запятая/кавычка) — **не коммитить**, чинить.
2. **Явное одобрение на push.** Покажи дифф (`git diff`) и пушь **только после явного «пушь» / «да»**. Никогда не пушить молча, без показанного диффа. Коммитить-в-`main`-напрямую можно (PR не обязателен), но push — только с подтверждения.
3. **Стоп-проверка скоупа.** Если правка задевает больше, чем просили — удаляет запись целиком, меняет не то поле, трогает несколько практиков вместо одного — **остановись и переспроси**. Лучше уточнить, чем выкатить лишнее на живой сайт.
4. **Коммит** — осмысленное сообщение на русском (как в истории репо), с трейлером `Co-Authored-By: Claude …`.

## Откат

Откатить выкат: `git revert <commit>` + push, либо взять файл из бэкап-тега `backup-pre-json-refactor-2026-05-30` (снимок до выноса данных в JSON). История `main` — полный лог изменений.
